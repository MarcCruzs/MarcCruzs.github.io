## Project Goals
As part of this team, we would be planning to participate through CPP Project Hatchery for funding. With that in mind, we will be following all requirements from Project Hatchery. Then the project plans to conduct literature review and then analysis of potential portable hardware that can be oriented towards machine learning applications. Then creating a test bed that includes various hardwares that will be ranked based on capabilities and any measurement discovered during literature review. After creating the test bed, would develop a machine learning model to test the test bed. Further testing would be conducted utilizing various models from other projects at CPP to ensure the test bed is capable of catering to the projects that occur at CPP. If the work is significant enough, then consideration of publishing work is in order. 

To sum up the goals:
- Follow Project Hatchery Requirements
- Conduct literature review and analysis on potential hardware to be used for AI applications
- Design and create a testbed to be used to measure inference time, power usage, and other metrics between different hardware
- Test the testbed on a developed machine learning model and other models that exist at CPP
- Potentially publish work depending on the significance and contribution of team members.

## Motive
Through various trade studies conducted at the UAV Lab, there has been a noticable lack of strong connections/reasoning of purchasing hardware that is bought for machine learning performance. Then for trade studies that attempted to do proper comparisons of hardware noticed different usage of metrics between various groups and manufacturers. Leading to harder comparison of hardware and difficult to find hardware that has the optimal amount of performance, while sparing as much power usage to be more effective for portable operations.

## Brief Summary
Research and analysis on various portable hardware capability for machine learning applications. Doing analysis on current methedologies utilized to measure a hardware capability. From this research, creating a test bed that incorporates various hardwares of various capability to act as a test bed for machine learning applications designed for "on the go" operations. The test bed acts as a way to conduct proper trade studies between hardware with manufacturers using different metrics. Also can be used to determine a way to do experiments to determine what is the most optimal hardware to have full capability, while ensuring it is the cheapest & power efficient as possible. The end result is getting an inference time with a hardware's specs allows a project to determine how fast their model is. This information allows the project to have better educated purchases of their own hardware.

## Target Audience
Essentially for any project planning to do have a machine learning application that requires to be portable on a portable vehicle or isolated area. The project idea was created due to weak trade studies in the CPP UAV Lab, and supposed to be a solution to getting the most efficient hardware performance and power wise.

## How is the idea innovative and better than current ideas and better than current ideas/implementation
The idea was created in a way that allows it to open the door for other research to be conducted in an efficient manner. The possibility of being able to rely on physical testing via the test bed will allow more evidence on what a project may need. Instead of making assumptions on what may be the required project, that may lead to buying hardware that might be overkill for the case use or buying a different hardware as it might be underperforming.
